\section{Setup}
This run keeps the same revisit-termination rule but increases the training budget from 500 episodes to 5000 episodes.

Core settings:
\begin{itemize}
\item Termination: goal reached or revisit of any previously visited state in same episode
\item Potential: Manhattan PBRS
\item Discount: $\gamma=0.99$
\item Rewards: step $=0.0$, goal $=1.0$
\item Episodes: 5000, Runs: 12
\item Validation: every 25 episodes (plus episode 1), 30 rollouts
\end{itemize}

\section{Interpretation}
Increasing training iterations alone did not recover success under this termination rule (all conditions remained at 0 validation success). This supports the view that the termination design, not only training budget, is the primary bottleneck.
